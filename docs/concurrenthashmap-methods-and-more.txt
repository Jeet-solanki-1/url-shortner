Here’s a complete reference for the main functions and methods of ConcurrentHashMap in Java, along with examples.
ConcurrentHashMap is part of java.util.concurrent and provides thread-safe operations without locking the entire map.

Key Points

Package: java.util.concurrent
Thread-safe: Multiple threads can read/write without external synchronization.
Nulls: Does not allow null keys or null values.
Performance: Uses internal segmentation to reduce contention.

Example: Using ConcurrentHashMap
Javaimport java.util.concurrent.ConcurrentHashMap;

public class ConcurrentHashMapDemo {
    public static void main(String[] args) {
        // Create a ConcurrentHashMap
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

        // put() - Add entries
        map.put("Apple", 10);
        map.put("Banana", 20);

        // putIfAbsent() - Add only if absent
        map.putIfAbsent("Orange", 30);

        // get() - Retrieve value
        System.out.println("Apple count: " + map.get("Apple"));

        // replace() - Update value
        map.replace("Banana", 25);

        // remove(key, value) - Conditional remove
        map.remove("Orange", 40); // Won't remove
        map.remove("Orange", 30); // Will remove

        // computeIfAbsent() - Compute value if missing
        map.computeIfAbsent("Mango", k -> 50);

        // computeIfPresent() - Update if present
        map.computeIfPresent("Apple", (k, v) -> v + 5);

        // merge() - Merge values
        map.merge("Mango", 10, Integer::sum);

        // forEach() - Iterate
        map.forEach((k, v) -> System.out.println(k + " => " + v));

        // size()
        System.out.println("Map size: " + map.size());
    }
}


Thread-Safety Note
Unlike HashMap, you can safely use ConcurrentHashMap in multi-threaded environments without external synchronization.
However, compound operations (like check-then-act) should still use atomic methods like putIfAbsent, compute, or merge to avoid race conditions.
